Introduction

  • Popular high level data model and is an approach to designing semantic conceptual schema of a database
  • Allows us to describe the data involved in a real world environment in terms of objects and their relationships, which are widely used in design of database
  • Provides preliminary concepts or ideas about the data representation which is later modified to achieve final detailed design

Entity

  • Object in a real world or some idea or concept which can be distinguished from other objects
  • Has independent existence
  • Each entity belongs to an entity type that defines the structure
  • Entity set is a collection of similar objects

Concepts used in ER

  • Attribute reflects a property of an object or entity

  • Key is an attribute of an entity type whose value can uniquely identity an entity in set
  • Relationship is the association between entities
  • Domain of an attribute is the set of all possible values that attribute can take

EER

  • EER stands for Extended ER modeling or Enhanced ER modeling
  • EER includes object oriented concepts such as inheritance
    • Specialization and generalization
    • Attribute and Relationship Inheritance
    • Subclasses and super classes

Subclasses in EER

  • An entity type may have additional sub groupings
  • Subclass member is the same entity in a distinct specific role
  • An entity cannot exist in the database merely by being a member of the subclass
  • It must also be a member of the superclass
  • A member of the superclass can be optionally included as a member of any number of itâ€™s subclasses

Specialization

  • Process of defining a set of subclasses of a superclass
  • Here secretary cannot be technician and vice versa â†’ disjoint
  • d in circle means disjoint while o means overlapping

Generalization

  • Reverse of the specialization process.
  • Several classes with common features are generalized into a superclass

Constraints

  • We have two types of constraints
    • Disjoint-ness Constraint and
    • Completeness Constraint
  • Hence we have 4 types of specialization or generalization
    • Disjoint total
    • Disjoint partial
    • Overlapping total
    • Overlapping partial
  • Generalization is usually total because the superclass is derived from the subclasses
