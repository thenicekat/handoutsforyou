Abstraction

  • It denotes the essential characteristics of an object that distinguish it from all other kinds of objects and thus provide crisply conceptual boundaries from the POV of the user
  • It focuses on the outside view of an object
  • Allows us to layer semantics of complex systems. From outside car is a single object but inside, car has multiple sub systems: steering, brakes, sound etc.

Encapsulation

  • Encapsulation is most often achieved by hiding information and secrets of an object that do not contribute to it’s essential characteristics
  • Encapsulation is the mechanism that binds together code and the data it manipulates, and keeps both safe from outside interference and misuse.
  • One way to think about encapsulation is as a protective wrapper that prevents the code and data from being arbitrarily accessed by other code defined outside the wrapper.
  • Access to the code and data inside the wrapper is tightly controlled through a well-defined interface.
  • The public keyword is an access modifier, which allows the programmer to control the visibility of class members.
  • When a class member is preceded by public, then that member may be accessed by code outside the class in which it is declared. (The opposite of public is private, which prevents a member from being used by code defined outside of its class and protected means only it's subclasses can access it)
  • In this case, main( ) must be declared as public since it must be called by code outside of its class when the program is started.
  • The keyword static allows main( ) to be called without having to instantiate a particular class instance.
  • This is necessary since main( ) is called by the Java Virtual Machine before making any objects.
  • The keyword void simply tells the compiler that main( ) does not return a value.
  • As you will see, methods may also return values
  • When a member of the class is made private, then the member can only be accessed by other members.
  • Constructors exist and have the same name as the class. Default Constructor initializes all variables to 0. Use constructor overloading for constructors of different types
  • Constructor chaining is when you can use another constructor from another constructor, the more specific constructor is called from a less specific constructor.
  • This keyword can be used in other cases also
  • We can use java's toString method to directly convert the object to string, but you might also have to overload the existing method to make it work however you want. The toString method is called if you directly print an object i.e. overloads the print function
  • Use @Override when overriding methods. Object instanceof Class returns true or false depending on the object
  • The iterator class can help iterate over all elements of the class
  • When we have a static method, it can be made to run without making a new object from the class
  • Static methods have some restrictions
    • They can only directly call other static methods of their class
    • They can only directly access static variables of their class
    • They cannot refer to this or super in any way.

Modularity

  • It is partitioning a program into modules that can be compiled separately, but they have connections with other modules.
  • Used to reduce complexity. Reduce software cost by allowing modules to be designed and revised independently.

Hierarchy

  • Is a ranking or ordering of abstractions
  • Two hierarchies: is-a and part-of
  • Inheritance is the most important IS-A hierarchy, it is the process by which an object gets the properties of another object

Inheritance

  • Inheritance is used to reuse already existing classes with some modifications. Java does not allow multiple inheritances. Use the extends keyword: subclass extends superclass
  • We use super() to call the constructor of the superclass
  • Java automatically invokes a super() constructor without parameters before the first line of the subclass constructor
  • The superclass can have a private method by which the child can make public
  • a final class cannot be subclassed
  • Abstract classes have method headers, but no body. Similar to prototyping
  • A subclass cannot access private methods of the superclass but only the protected ones
  • In inheritance, the constructors follow their execution in order of derivation i.e., from superclass to subclass
  • When a method in the subclass and the superclass have the same name and type sign, the one in the subclass is said to override the one in the superclass
  • Abstract classes are those which don’t have complete specifications
  • NOTE: When we use inheritance and store, say child class object in the parent class, even though the functions happen from the child class, the variables of the parent class gets executed

Polymorphism

  • The ability of an object to take many forms. Allows us to perform the same action in multiple ways
  • Method Overloading - where you create new functions with same name but different signature
  • Method Overriding - where you override the already existing function
  • Two Types
    • Compile Type Polymorphism- Static Polymorphism
      • Function or Operator Overloading
    • Run time Polymorphism- Dynamic Method Dispatch
      • When an overridden method is called through a superclass reference, Java determines which version of that method to execute based on the type of object referred to when the call occurs.
      • Upcasting is done when the parent class’s reference variable refers to the object of the child class. This is used in Runtime Polymorphism.

Typing

  • Typing is the enforcement of the class of an object such that objects of different types may not be interchanged or at the most may be interchanged in very restricted ways
  • Strongly Typed - Type conformation is strictly enforced.
  • Each type of data is predefined as part of the language and all constants must be described with one of the data types
  • Early detection of bugs makes it effective. Helps document programs
  • Compilers generate efficient code
  • Java and C++ are strongly typed
  • Weakly typed makes conversions between unrelated types implicitly
  • Perl is weakly typed
  • Untyped - Violation of type conformance may not be known until run time example Small Talk
    • Ada: Strongly typed and static binding.
    • Smalltalk: Untyped and dynamic.
    • Java, C++: Strong typed and support dynamic binding

Binding

  • Refers to the time when names are bound to object types
  • Static → Type of object pointed by reference is fixed at time of compilation
  • Dynamic → Type of object pointed by reference is not fixed and not known until runtime

Composition and Aggregation

  • Aggregation is where another class has an object of another class more like a has-a relationship
  • Composition is one of the fundamental concepts in object-oriented programming. It describes a class that references one or more objects of other classes in instance variables. This allows you to model a has-a association between objects
  • Aggregation implies a relationship where the child can exist independently of the parent. For example, Bank and Employee, delete the Bank and the Employee still exist. whereas Composition implies a relationship where the child cannot exist independently of the parent. Example: Human and heart, heart don’t exist separate to a Human
  • Composition is a strong association
// Java Program to Illustrate Difference between
// Aggregation and Composition

// Importing I/O classes
import java.io.*;

// Class 1
// Engine class which will

// class will have a field

class Engine {

// Class 2
// Engine class
final class Car {

	// For a car to move,
	// it needs to have an engine.

	// Composition
	private final Engine engine;

	// Note: Uncommented part refers to Aggregation
	// private Engine engine;

	// Constructor of this class
	Car(Engine engine)
	{

		// This keywords refers to same instance
		this.engine = engine;
	}

	// Method
	// Car start moving by starting engine
	public void move()
	{

		// if(engine != null)
		{
			// Calling method for working of engine
			engine.work();

			// Print statement
			System.out.println("Car is moving ");
		}
	}
}

// Main class
class GFG {

	{

		// Making an engine by creating

		// passing a engine instance as an argument

Concurrency

  • Refers to potentially parallel execution of parts of a computation
  • Components of a program may be executed sequentially or parallelly
  • Helps in solving problems where we need to handle multiple events simultaneously
  • Java has multithreaded

Persistence

  • When an application is creating an object, the scope of object is limited to the application life cycle
  • When application terminates, the object also terminates
  • Object is being stored temporarily in main memory
  • To keep objects alive we use something called persistence storage
  • The concept of maintaining the state of the object even after it’s creator ceases to exist is called persistence
  • Can be implemented using text files, databases, object oriented database

Cohesion

  • Cohesion in Java is the Object-Oriented principle most closely associated with making sure that a class is designed with a single, well-focused purpose
  • More cohesion is better
