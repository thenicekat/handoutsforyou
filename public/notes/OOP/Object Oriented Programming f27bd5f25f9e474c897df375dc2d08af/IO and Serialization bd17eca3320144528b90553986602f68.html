File

  • Does not operate on streams but rather operates directly on files
  • Describes the properties of a file
  • File object is used to manage properties of the file
  • A directory in java is just a file with one extra property i.e. list() of files

Constructors

  • File(String directoryPath)
  • File(String directoryPath, String filename)
  • File(File dirObj, String filename)
  • File(URI uriObj)
  • If you want to use \ in the file path, you need to put in two backslashes \\
File f1 = new File("/"); // includes directory path only
File f2 = new File("/",“f1.txt"); // includes the path and the filename
File f3 = new File(f1,“f1.txt");
// the file path assigned to f1 and a filename; f3 refers to the same file as f2

Obtaining Properties of a File

  • getName() gives us name of the file
  • getParent() returns name of parent directory
  • exists() returns true of false depending on if file exists or not
  • isFile() tells if it’s a file or a directory
  • isAbsolute() tells us if the path is absolute or relative
  • renameTo(String newName) renames files and returns true or false
  • delete() deletes the directory(only if empty) or the file

Directory

  • Use isDirectory() to check if it’s a directory or not
  • Then use list() method to get list of files in that
    • String[] list() → list of files
    • String[] list(FileNameFilter FFObj) → can be used to limit the number of files returned by the list() method to include only filtered ones
  • FilenameFilter defines only a single method, accept( ), which is called once for each file in a list. Its general form is given here: boolean accept(File directory, String filename)
  • We can also use the mkdir() and mkdirs() to create directory
    • mkdirs is used to create directory even if the parents don’t exist i.e. it creates parents as well

Stream

  • Path along which data flows
  • Has a source and a destination and these facilitate transporting data from one place to another. Without streams, it is not possible to move data in java
  • A stream is a logical entity that either produces or consumes data
  • There are 2 types of streams,
    • InputStream
    • OutputStream
  • Can also be classified as byte streams and character streams
  • Bytestreams → represent data in the form of individual bytes, if a class name has Stream in the end it’s a ByteStream
  • Character Streams → represent data as characters of each 2 bytes, they can always store and receive data but only in the form of characters
  • If a class name ends with Reader or Writer, it’s a TextStream

Input Stream Classes

  • Used to read 8 bit bytes
  • It’s an abstract class
  • Defines methods for performing
    • Reading bytes
    • Closing Streams
    • Marking positions in streams
    • Skipping ahead in a stream
    • Finding number of bytes in a stream

Output Stream Classes

  • Writing Bits
  • Closing Streams
  • Flushing Streams
import java.io.*;

public class CopyFile {
    public static void main(String[] args) {
        File f1 = new File(args[0]);
        File f2 = new File(args[1]);

        try {
            FileInputStream fis = new FileInputStream(f1);
            FileOutputStream fos = new FileOutputStream(f2);

            int temp = 0;
            try {
                while ((temp = fis.read()) != -1) {
                    System.out.println("::>" + (char) temp);
                    fos.write(temp);
                }

                System.out.println("Copy Successful");
            } catch (IOException e) {
            }

        } catch (FileNotFoundException e) {

        }
    }
}

Buffered Streams

  • Buffered Stream extends a filtered Stream by attaching a buffer to the stream and ensuing operations can take place on more than one byte
  • Skipping, marking and resetting of stream becomes possible

Buffered Input Stream

  • Wraps any InputStream into a buffered stream to improve performance
  • Has two constructors
    • BufferedInputStream(InputStream in)
    • BufferedInputStream(InputStream in, int bufSize)
  • Default buffer size is 8192 bytes
  • Has mark and reset methods in addition to other methods
import java.io.*; 
public class BufferedInputStreamExample{ 
	public static void main(String args[]){ 
		try{ 
			FileInputStream fin=new FileInputStream("input.txt"); 
			BufferedInputStream bin=new BufferedInputStream(fin); 
			int i; 
			while((i=bin.read())!=-1){ 
				System.out.print((char)i); 
			} 
			bin.close(); 
			fin.close(); 
		}
		catch(Exception e){System.out.println(e);} 
	} 
}

Buffered Output Stream

  • Same as output stream but has flush() to ensure that data buffers are written to the stream being buffered
  • Same type of constructors as before
  • BufferedOutputStream is waay faster than normal FileOutputStream

RandomAccessFiles

  • These classes implement both the DataInput and DataOutput interfaces and therefore can be used to read and write files
  • You can use new RandomAccessFile(”data.txt”, rw);
  • It can be used for both reading and writing
  • We have skipBytes(int n), seek(long pos), getFilePointer() as methods of this class

Character Streams

  • Store and retrieve data in the form of only characters
  • ReaderClass → streams character input
  • WriterClass → streams character output

FileWriter

  • The constructors of FileWriter are provided below:
    • FileWriter(Sting filePath)
    • FileWriter(String filePath, boolean append)
    • FileWriter(File fileObj)
    • FileWriter(File fileObj,boolean append)
  • Here, filePath is the full path name of a file, and fileObj is a File object that describes the file. If append is true, then output is appended to the end of the file.
  • Note: An IOException is thrown when you make an attempt to open a readonly file.

FileReader

  • It is useful to read data in the form of characters from a text file.
  • It has two most commonly used constructors:
    • FileReader(String filePath)
    • FileReader(File fileObj)
  • Note: Either of the above throw a FileNotFoundException. filePath is the full path name of a file and the fileObj is a File object that describes the file

BufferedWriter

  • It buffers output and improves performance of writing by reducing the number of times data is physically written to the output device
  • Has two constructors
    • BufferedWriter(Writer outputStream): It creates a buffer with default size
    • BufferedWriter(Writer outputStream, int bufSize): It creates a buffer with the specified size.

Serialization

  • Serialization is the process of storing object contents into a file
  • The stored objects can later be retrieved and used when needed through a process called Deserialization
  • The class whose contents are to be stored in the file should implement Serializable interface
  • Also useful in implementing RMI i.e. Remote Method Invocation, this allows an object on one machine to invoke a method on another machine
  • The sender serializes the object and sends
  • NOTE: Static and transient variables cannot be serialized

Serializable Interface

  • Defines no members
  • Simply used to indicate that class maybe serialized
  • If a class is serializable, all it’s sub classes are serializable
  • If it’s not serializable and we try to serialize it, we get a NotSerializable Exception

ObjectOutput and ObjectInput Interfaces

  • WriteObject() is one of the methods of ObjectOutput and this takes in the object to be serialized and gives IOException if the arguments are not formatter properly
  • ReadObject() is one of the methods of ObjectInput and gives out the object after deserialization and can give rise to IOException or the ClassNotFoundException

ObjectOutputStream and ObjectInputStream

  • Extends OutputStream and implements ObjectOutput Interface
  • Writes objects as a stream
  • Constructor → ObjectOutputStream(OutputStream outStream) throws IOException
  • Same with ObjectInputStream
  • Most important method of OOS is writeObject(Object obj) and for OIS it’s readObject(Object obj)
import java.io.*;
import java.util.*;

class Employee implements Serializable {
    private int id;
    private String name;
    private float sal;
    private Date obj;

    Employee(int i, String n, float s, Date d) {
        id = i;
        name = n;
        sal = s;
        obj = d;
    }

    void display() {
        System.out.println(id + "\t" + name + "\t" + sal + "\t" + obj);
    }
}
